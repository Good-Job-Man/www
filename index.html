1. 구매가 통계의 시작점구매자가 결제를 완료하면 purchase와 PURCHASE_DETAIL이 생성된다. 통계는 이 상세 데이터의 quantity와 unit_price를 기준으로 계산한다.

4. 실시간/외부 API는 공통 인프라WebSocket 상담, Naver 표지, Kakao 로그인, Brevo 인증, Toss 결제가 기능별 흐름과 연결된다.

04 · DATABASE MAP

박지성 정리 DB 관계도 원본 + 재정리

원본 ERD를 그대로 인용하고, 발표에서 빠르게 설명할 수 있도록 도메인별 관계를 다시 묶었다.

사용자/권한 축

COSMIC_USER, USER_REGISTRATION, BASE_ADMIN, VENDOR, VENDOR_REGISTRATION이 로그인 주체와 권한을 구성한다.

상품/판매 축

BOOK → STOCK_IN → PRODUCT_SALE 구조로 도서 원천 데이터, 입고, 협력업체 판매 상품이 연결된다.

구매/결제 축

purchase와 PURCHASE_DETAIL은 정회원 주문의 원천이고, ORDERS는 Toss 결제 상태를 연결한다.

비회원 축

GUEST_USER → GUEST_PURCHASE → GUEST_PURCHASE_DETAIL 구조로 비회원 정보와 주문 상세를 분리한다.

상담/문의 축

PARTICIPANT가 회원·업체·관리자를 통합 식별하고, QNA_CHAT/QNA_MAIL이 상담 기록을 저장한다.

도메인대표 테이블핵심 관계
회원/관리자/업체COSMIC_USER, BASE_ADMIN, VENDOR, VENDOR_REGISTRATION각 로그인 주체가 기능 접근 권한과 연결된다.
도서/상품BOOK, STOCK_IN, PRODUCT_SALEBOOK.id가 입고와 판매 상품으로 이어진다.
정회원 구매purchase, PURCHASE_DETAIL, ORDERS구매 마스터와 상세가 결제 상태 및 통계로 연결된다.
비회원 주문GUEST_USER, GUEST_PURCHASE, GUEST_PURCHASE_DETAIL비회원 원천 정보와 영수증, 품목 상세를 분리한다.
리뷰review_book, review_user도서별 평균 별점과 사용자 리뷰를 분리한다.
상담/문의PARTICIPANT, QNA_CHAT, QNA_MAIL통합 통신 식별자 p_id로 실시간/비대면 문의를 관리한다.
인증TB_EMAIL_AUTH이메일 인증 코드, 만료 시간, 인증 여부를 저장한다.

실시간 상담 · QnAChat

Owner · 박지성
기능개요

유저, 관리자, 업체가 각각 다른 테이블로 관리되더라도 PARTICIPANT 테이블에서 하나의 통신 식별자 p_id로 묶어 실시간 상담 채널을 구성한다.

모든 페이지에서 접근 가능한 모달 형태로 배치되며, WebSocket을 통해 메시지를 즉시 라우팅하고 QNA_CHAT에 영구 저장한다.

통신 방식WebSocket

브라우저와 서버 사이의 전이중 통신

식별 기준PARTICIPANT.p_id

회원·업체·관리자를 하나의 통신 주체로 통합

저장 기준QNA_CHAT

sender_pid / receiver_pid / message / is_read 저장

화면구성 / 구조

  • Floating Action Button: 우측 하단 고정 상담 버튼
  • Cosmic Side Panel: 우측 슬라이드 패널로 채팅 UI 표시
  • User Mode: 관리자와 1:1 상담 인터페이스
  • Admin Mode: 좌측 대원 목록 + 중앙 대화 세션
  • Message Stream: 발신/수신 말풍선, 시간, 자동 스크롤
  • Input Footer: Enter 또는 전송 버튼으로 메시지 송출

FEATURE DETAIL

LoginInterceptor · 세션 기반 인증/인가

Owner · 박지성
기능개요

Cosmic Library의 모든 HTTP 요청을 Controller 도달 전에 가로채는 보안 모듈이다.

세션 기반 인증과 역할 기반 인가를 중앙에서 검증하여 미인가 접근을 차단하고, 목적지에 맞는 로그인 화면이나 대시보드로 라우팅한다.
